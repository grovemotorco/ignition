Ignition uses an event pipeline for observability. Every significant action during a run emits a structured event that reporters, the dashboard, and log sinks consume.

## EventBus

The `EventBus` is the central event dispatcher:

```typescript
import { EventBus } from "@grovemotorco/ignition"

const bus = new EventBus()

const unsubscribe = bus.on((event) => {
  console.log(`[${event.type}]`, event.timestamp, event)
})

// Pass to runRecipe
const summary = await runRecipe({
  // ...
  eventBus: bus,
})

unsubscribe()
```

Each event bus generates a unique `runId` (12 hex chars) for correlation. All events within a run share this ID.

## Event types

Eight event types cover the full lifecycle:

### Run events

| Event          | Description                                                 |
| -------------- | ----------------------------------------------------------- |
| `run_started`  | Run begins. Payload: `mode`, `errorMode`, `hostCount`       |
| `run_finished` | Run ends. Payload: `durationMs`, `hasFailures`, `hostCount` |

### Host events

| Event           | Description                                                                                |
| --------------- | ------------------------------------------------------------------------------------------ |
| `host_started`  | Host execution begins. Payload: `host`                                                     |
| `host_finished` | Host execution ends. Payload: `host`, `ok`, `changed`, `failed`, `durationMs`, `cancelled` |

### Resource events

| Event               | Description                                                                                                   |
| ------------------- | ------------------------------------------------------------------------------------------------------------- |
| `resource_started`  | Resource execution begins. Payload: `resourceType`, `resourceName`                                            |
| `resource_finished` | Resource execution ends. Payload: `resourceType`, `resourceName`, `status`, `durationMs`, `error`, `cacheHit` |
| `resource_retry`    | A retryable error triggered a retry. Payload: `resourceType`, `resourceName`, `phase`, `error`, `durationMs`  |
| `resource_output`   | Streaming output chunk. Payload: `resourceType`, `resourceName`, `stream` (`"stdout"` or `"stderr"`), `chunk` |

## Event structure

Every event extends `BaseEvent`:

```typescript
interface BaseEvent {
  type: EventType
  timestamp: string // ISO 8601
  correlation: {
    runId: string
    hostId?: string
    resourceId?: string
    attempt?: number
  }
}
```

## Subscribing to events

Register listeners with `bus.on()`:

```typescript
const bus = new EventBus()

bus.on((event) => {
  switch (event.type) {
    case "resource_finished":
      if (event.status === "failed") {
        console.error(`FAILED: ${event.resourceType} ${event.resourceName}`)
      }
      break
    case "run_finished":
      console.log(`Run complete in ${event.durationMs}ms`)
      break
  }
})
```

`bus.on()` returns an unsubscribe function.

## NDJSON streaming

`NdjsonStream` serializes events as newline-delimited JSON:

```typescript
import { EventBus, NdjsonStream } from "@grovemotorco/ignition"

const bus = new EventBus()
const stream = new NdjsonStream({
  writeSync: (p: Uint8Array) => {
    process.stdout.write(p)
    return p.length
  },
})

bus.on(stream.listener)
```

Each line is a complete JSON object followed by a newline. This format is compatible with tools like `jq`:

```bash
jq -r ".type" ./logs/*.ndjson
```

## File log sink

`FileLogSink` powers the CLI `--log-dir` option and writes NDJSON events to timestamped log files:

```bash
ignition run deploy.ts @web --log-dir ./logs
```

This creates files like `logs/2026-03-02T14-22-08Z_a1b2c3d4e5f6.ndjson`.

## Building a custom event consumer

Example: send Slack notifications on failures:

```typescript
import { EventBus } from "@grovemotorco/ignition"
import type { LifecycleEvent } from "@grovemotorco/ignition"

function slackNotifier(webhookUrl: string) {
  return (event: LifecycleEvent) => {
    if (event.type === "run_finished" && event.hasFailures) {
      fetch(webhookUrl, {
        method: "POST",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify({
          text: `Ignition run failed after ${event.durationMs}ms`,
        }),
      })
    }
  }
}

const bus = new EventBus()
bus.on(slackNotifier("https://hooks.slack.com/..."))
```

## Event flow diagram

```
Recipe execution
  ├── run_started
  ├── host_started (per host)
  │     ├── resource_started
  │     ├── resource_output (streaming, 0..n)
  │     ├── resource_retry (on transient failure, 0..n)
  │     ├── resource_finished
  │     └── ... more resources
  ├── host_finished (per host)
  └── run_finished
```
